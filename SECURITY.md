# Security Policy

## Supported Versions

The following versions of TruthBounty API are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.0.1   | :white_check_mark: |
| < 0.0.1 | :x:                |

## Reporting a Vulnerability

We take the security of TruthBounty API seriously. If you believe you have found a security vulnerability, please report it to us as soon as possible.

**Please do not report security vulnerabilities through public GitHub issues.**

### Reporting Method

Please report any security vulnerabilities by sending an email to **security@truthbounty.io**.

When reporting a vulnerability, please include the following information:
- A description of the vulnerability.
- Steps to reproduce the issue (including any proof-of-concept code).
- The potential impact of the vulnerability.
- Any suggestions for mitigation or fixes.

### Our Response Process

1. **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
2. **Investigation**: We will investigate the issue and determine its severity.
3. **Update**: We will keep you informed of our progress.
4. **Resolution**: Once fixed, we will release a security advisory and a new version of the API.

## Disclosure Policy

We follow a coordinated disclosure policy. We ask that you give us a reasonable amount of time to fix the issue before making it public. In return, we will:
- Work with you to understand and resolve the issue.
- Provide credit to you in our security advisories (if you wish).

## Bounty Program

At this time, TruthBounty does not have a formal financial bug bounty program, but we are a community-driven project and deeply appreciate the efforts of security researchers in keeping the ecosystem safe. Outstanding contributions to project security will be recognized within our community.
